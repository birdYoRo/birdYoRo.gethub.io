PAT 1155 Heap Paths

还挺直接易懂的题目，可能是一边上网课一边打，居然打了40分钟，有点慢啊

  • 耗时：40min
  • 关键词汇： complete binary tree 完全二叉树
  • 题目大意：给一个完全二叉树的序列，从右到左列出所有根到叶子结点的路径，并判断是小根堆还是大根堆。
  • 注意点:
    1. 判断完全二叉树的一条路径是否完整只需检查它是否有左子树，没有左子树则停止遍历
    2. std::vector初始化可以以数列为参数，如vector<int>(arr,arr+n)作为参数传入，或者vector<int> ar(in,in+5);来初始化；
    3. 非递减非递增数列，可以用递增个数+相等个数或递减个数+相等个数与数列总个数比较；
    4. 用顺序存储结构中二叉树左右结点下标与根节点的关系，根结点下标i（从0开始编号），左节点i*2+1，右节点i*2+2。
  • PS:
    • 回过头来想想，好像我从序列的最后一个结点开始列路径，并用visited数组记入这个结点是否被输出，如果输出过了，肯定就不是叶子节点了;
    • 涉及堆的题目好像从后往前思考比较好。

*PAT 1151 LCA in a Binary Tree

好题！

但是这题用while(scanf(…)!=EOF)输入会卡一个测试用例，不知道为什么。

  • 耗时：花了40多分钟打了只对了四个用例，然后看来别人的blog重写了一遍。
  • 关键词汇：
    • ancestor：祖先
    • LCA(Least Common Ancestors)：最近公共祖先
  • 题目大意：给一个二叉树的后序遍历序列和先序遍历序列，在给出结点对若干，对每个结点对计算其LCA；
  • 注意：
    • 题目的序列长度是<=10000的，对于我初始的做法，是处理先序后序两个序类得到每个结点顺序存储的下标号index，但是若干题目给的二叉树不是一棵完美二叉树的话，而是一棵每个结点只有右子树的树的话，那么最大的index就可能大于2^10000，就算我开long long也无法放下这么大的数据。因此这题必须在递归中处理。
    • 如果一个结点是两个结点的最近公共结点的特征是在一个位于根节点左子树中，一个位于根节点的右子树中，如果两个结点都在左子树中那就进入左子树递归，都在右子树中则进入右子树递归；
    • 可能a在左子树b在右子树，也可能a在右子树b在左子树，判断时不要遗漏；
    • 遍历时注意根节点下标的值；
    • map存放的数据可以不用insert直接赋值，对于基本数据类型也可以通过键直接查找，如过没有目标键则返回默认为0；
    • 如果使用顺序结构建图一定要确定题目是否指明complete binary tree！！！

PAT 1150 Travelling Salesman Problem

简单题，一个毛概视频的时间打完了，然后漏了好多判断，debug了半天。。。

  • 耗时：25min
  • 题目大意：给一个图的信息，构建一个图，然后给出一条路径信息，计算路径长度并且判断是否是一个圈，最后给出最短的一条路径
  • 注意：
    • 直接用邻接矩阵构图就可以；
    • 注意判断not a ts cycle的点，要长度大于总结点数+1，并且首位结点相同，并且存在路径；
    • 我用set来记录已算结点，题目没给路径范围所以没有定义数组存放结点，在输入时直接计算了。

PAT 1146 Topological Order

图论的简单题，邻接矩阵+拓扑排序建图

  • 耗时：15min
  • 关键词汇：
    • directed acyclic graph（DAG）：有向无环图 //题目中是direct graph
    • Topological Order：拓扑次序
  • 题目大意：给一个有向图的边的序列，再给出若干结点序列，最后输出其中不是拓扑序列的序号
  • 注意:
    • 因为每次都要使用一个初始的入度值，所以我直接用vector<EleType> arr1((vector<EleType>)arr2)的方式初始化一个和入度值的副本，也可以用c库的memcpy；
    • 结点范围为<=1000，而变的范围为<=10000，是一个稀疏图，邻接表或者链式前向星构图是更好的选择，但是结点数没有太多，我还是选择了邻接矩阵，方便嘿嘿；
    • 要求判断的序列从0开始排序。

*PAT 1145 Hashing - Average Search Time

第一次做hash函数和冲突解决的题目，看了解析做了，解决冲突的方法都记不太清了

  • 耗时：–
  • 关键词汇：
    • Quadratic probing (with positive increments only) 平方探测法（只带正增量）
    • collisions 冲突
  • 题目大意：给了一个序列要放到hash表中存储，哈希表的大小要是大于等于已给MSize的最大素数，接着给出一个数组，从hash表中找是否存在数组的元素，并记录查找次数取平均值；
  • 注意：
    • 该题的哈希函数是H(key)=key%Tsize，使用平方探测法解决冲突得到的哈希函数为H(key)=(key+i*i)%Tsize，i的取值范围为[1,Tsize)，因为题目指明了只带正增量。如果没有声明按我查到的方法，i的取值还可以是1^2 -1^2, 2^2, -2^2, …… , i^2, -i^2，且 q<= [ TSize/2 ] 循环试探下一个存储地址；
    • 按这题的解法，在一个数试探完所有增量i后，还要对搜索次数加1才行（应该是代表下一个i已经不能用于试探），我也不是很清楚为什么要加这个1，难道是一种默认的规则？等数据结构看到再回过头来看看算了。

* PAT1143 Lowest Common Ances

和PAT1151几乎一模一样的题目，但是时间卡的很严，需要使用unodered_map或者unordered_set才可以卡过去，而且我看来柳婼的做法，我不太认同她的做法，应该是这题的数据比较弱，她的那种做法才可以AC

  • 耗时：30min //主要有个案例一直超时
  • 关键词汇：
    • binary search tree (BST) 二叉搜索树
  • 题目大意：给一个长度为n的BST前序遍历序列，接着给出m个结点对，求其LCA
  • 注意：
    • 因为是BST所以可以对给出的前序遍历序列排序，得到其中序遍历序列，当然这么做的前提是前序序列中的数必须是distinct的；
    • 对比PAT1151，将结点对a，b放入参数，在函数中动态的求其对应序号，这题的预留时间不允许这么做，因为a，b的对应中序遍历的序号是默认不变的，所以可以设置为全局变量只进行一次计算！
    • 如果是使用map去存放中序遍历信息的话，map容器会默认对键first值进行排序，而对于unordered_map（unordered_map，它是一个关联容器，内部采用的是hash表结构，拥有快速检索的功能。），则unordered_map拥有更好的检索功能;
    • 看见有些人用unordered_set做，与unordered_map同理。
